Projet 2 — Requêtez une base de données avec SQL

Accueil • ~40 heures

Avant de démarrer, nous vous conseillons de :

  • Lire le scénario en entier, chaque section du projet et les documents fournis ;
  • Préparer une liste de questions pour la première session de mentorat.

Scénario

Vous arrivez en mission de consulting comme Business Intelligence Analyst chez Dat’Assur Habitation, société d’assurance habitation pour particuliers.

Le directeur général souhaite différencier l’entreprise en s’appuyant sur les données pour analyser le portefeuille actuel de contrats et devenir data-driven.

Mail de bienvenue — Pascal Hernandez (CTO)

Objet : Bienvenue

But du projet : revoir les prix des contrats afin d’être plus compétitifs, avec une tarification lisible selon le profil (localisation, propriétaire/locataire, maison/appartement…).

Votre première mission : faire un état des lieux des contrats actuels.

Pièce jointe : Plan d’action.

À valider avec votre mentor avant d’aller plus loin :

  • Les données sont chargées dans la base ;
  • Le dictionnaire de données est complété (typologies) ;
  • Le modèle d’aide aux requêtes SQL est rempli.

Une semaine plus tard — Suite du projet

Objet : Dat’Assur Habitation — Suite du projet

Félicitations, la méthodologie de requêtage convient ! À présent :

  • Écrire les requêtes SQL pour extraire les données nécessaires ;
  • Réaliser les analyses restantes (cf. compte-rendu en PJ) — vous pouvez écrire directement les requêtes SQL ;
  • Envoyer les résultats et la requête associée dans un document PDF (utiliser des alias pour la lisibilité) ;
  • Rédiger la présentation (méthodologie, étapes, rappel des résultats) en ≤ 10 slides PDF.

Objectif : se prononcer sur l’état de la standardisation des tarifs à date. Une analyse poussée n’est pas attendue ; les résultats des requêtes suffisent, accompagnés d’une description claire de la démarche.

Pièce jointe : Compte rendu de réunion.

Livrables attendus

  • Document technique (PDF, 12 pages max) avec :
    • Capture d’écran de la base de données chargée ;
    • Exploration des types + capture du dictionnaire de données ;
    • Capture du modèle d’aide aux requêtes rempli pour 3 demandes d’analyses.
  • Liste complète des analyses (n° 1 à 12) avec résultats associés.
  • Présentation (≤ 10 slides PDF) : méthodologie, étapes, rappel des résultats.

Nommage recommandé (dossier zip Titre_du_projet_nom_prénom) :

  • Nom_Prenom_1_documentation_technique_mmaaaa
  • Nom_Prenom_2_liste_analyses_mmaaaa
  • Nom_Prenom_3_presentation_mmaaaa

Exemple : Dupont_Jean_1_documentation_technique_012022

Soutenance

  • 15 min — Présentation des livrables : base et exploration, méthodologie, réponses aux analyses ;
  • 10 min — Discussion (l’évaluateur joue Pascal, CTO) : méthode, SQL, fonctions, explications des résultats, notions de dimensions / indicateurs / agrégation ;
  • 5 min — Debrief hors rôle.

Votre présentation doit durer 15 min (± 5 min). En-dessous de 10 min ou au-dessus de 20 min, elle peut être refusée.

Compétences évaluées

  • Charger des données dans une base de données ;
  • Rédiger des scripts simples en SQL pour aider à la prise de décision business.
